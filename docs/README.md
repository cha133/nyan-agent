## 需求开发时期的文档约定

**默认：小需求不建临时文档。** 直接改代码，并在改动触及产品或架构边界时更新本目录和/或根 `AGENTS.md`。

**仅当开发者明确说这是复杂/大型需求时**，才在 [`.agents/docs/`](../.agents/docs/) 建立临时工作文档（例如范围、方案、状态、跨会话交接）。不要因为任务稍长就默认开一套临时文档。

复杂需求完成后：

1. 把需要长期保留的决策沉淀进本目录（及必要时根 `AGENTS.md`）。
2. 删除该需求在 `.agents/docs/` 下的临时文件。

`.agents/docs/` 不是长期事实源，也不提交进度流水账作为产品基线。无活动大型需求时，该目录仅保留 `.gitkeep`。
